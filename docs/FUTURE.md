# Future Roadmap (未来改进计划)

以下是针对 `Artistic Nav` 项目的未来改进建议和优化方向，旨在提升性能、用户体验和可维护性。

## 1. 功能增强 (Features)

*   **多用户/团队协作系统**
    *   目前仅支持单管理员。未来可引入 RBAC (基于角色的访问控制)，支持多人协作管理链接库。
*   **智能标签与搜索增强**
    *   引入 AI 语义搜索 (Vector Search)，允许用户通过自然语言描述查找工具（例如：“找一个能生成配色的工具”）。
    *   自动抓取链接的元数据 (Meta Tags) 并自动生成标签。
*   **数据分析看板**
    *   统计链接的点击率 (CTR)、热门分类、用户访问来源。
    *   可视化展示，帮助管理员优化收录内容。
*   **浏览器插件**
    *   开发 Chrome/Edge 插件，允许管理员在浏览网页时一键收录到 Artistic Nav。

## 2. 性能与架构 (Performance & Architecture)

*   **数据库迁移 (PostgreSQL / MySQL)**
    *   目前使用 SQLite，适合中小规模。随着数据量增长，建议迁移到云数据库 (如阿里云 RDS)，配合 Prisma 很容易切换。
*   **边缘缓存 (Edge Caching)**
    *   利用 Vercel 或 Cloudflare 的边缘网络缓存 API 响应，减少数据库查询压力。
*   **图片处理优化**
    *   引入 Next.js `<Image>` 组件的更高级用法（如 `loader`），配合 OSS 的图片处理服务（裁剪、压缩、WebP 转换）。

## 3. 开发者体验 (DX)

*   **CI/CD 流水线**
    *   配置 GitHub Actions，实现代码提交后自动运行测试、构建 Docker 镜像并部署到阿里云。
*   **单元测试与 E2E 测试**
    *   增加 Jest (单元测试) 和 Playwright (端到端测试) 的覆盖率，确保核心功能（上传、登录、渲染）的稳定性。

## 4. UI/UX 极致打磨

*   **WebGL 性能优化**
    *   目前的 3D 地球和粒子效果在低端设备上可能较卡。可以引入 LOD (Level of Detail) 策略或在移动端自动降级。
*   **更丰富的主题系统**
    *   除了“日光/黑夜”，支持自定义配色方案或“赛博朋克”、“极简主义”等多种预设主题。
